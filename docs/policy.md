# Core Policy (Draft)

## Payouts
- Payouts are only allowed for eligible winners based on final standings.
- No payout is allowed without VERIFIED KYC and no active anti-cheat hold.
- Cash payouts are disabled when Prize Mode = GIFT_CARD.
- All payout actions must be audited.

## Gift Cards
- Gift-card awards are the default when Prize Mode = GIFT_CARD.
- Codes are stored securely and issued by admin action with an audit log.
- The payout UI should clearly indicate cash vs gift-card mode.

## Tournament Integrity
- Anti-cheat holds always block payouts.
- All tournament state transitions must be logged.

## Auditability
- Money actions: entry fees, refunds, payouts, ledger updates.
- Admin actions: approvals, rejections, edits, overrides.
- Tournament actions: creation, lock, cancel, complete.
